# Topic: Sequence-to-Sequence (Seq2Seq) Models

## 1) Formal definition (what is it, and how can we use it?)

A Sequence-to-Sequence (Seq2Seq) model is a type of neural network architecture designed to transform one sequence into another sequence. It is particularly useful when the input and output sequences may have different lengths.  The core idea involves two main components:

*   **Encoder:** The encoder takes the input sequence as input and processes it to create a fixed-length vector representation called the "context vector" or "thought vector." This vector is intended to capture the essence or meaning of the entire input sequence. It summarizes the input.

*   **Decoder:** The decoder takes the context vector generated by the encoder as input and uses it to generate the output sequence, one element at a time. The decoder "unpacks" the information in the context vector to produce the desired output.  Crucially, the decoder often incorporates a recurrent mechanism (like LSTM or GRU) to maintain state while generating the output sequence, thus allowing it to handle dependencies between elements in the output.

**How can we use it?**

Seq2Seq models are used whenever we need to transform one sequence into another, especially when the lengths differ. Common use cases include:

*   **Machine Translation:** Translating text from one language to another. The input sequence is the sentence in the source language, and the output sequence is the sentence in the target language.
*   **Text Summarization:** Generating a shorter summary of a longer text document.
*   **Chatbots/Dialogue Generation:** Responding to user input with a relevant and coherent response. The input is the user's message, and the output is the chatbot's reply.
*   **Speech Recognition:** Transcribing audio into text.  The input is the audio signal, and the output is the text transcription.
*   **Code Generation:** Generating code from a natural language description.
*   **Image Captioning:** Generating a textual description of an image (less common, but can be approached as a sequence generation problem).
*   **Question Answering:** Generating an answer based on a given question and context.

## 2) Application scenario

Let's consider the **Machine Translation** scenario. Suppose we want to translate English sentences into French.

*   **Input:** "Hello, how are you?" (English)
*   **Output:** "Bonjour, comment allez-vous ?" (French)

The encoder would process the English sentence word by word, generating a context vector that represents the meaning of the sentence. The decoder would then use this context vector to generate the French translation, also word by word. The decoder would learn to produce "Bonjour" as the first word, given the context and its internal state, and then "comment" and so on. The decoder also learns to recognize the end of the sequence so it stops at the appropriate time.  The model learns the mapping between English and French through a large dataset of parallel sentences.

## 3) Python method (if possible)

Using TensorFlow and Keras, we can create a simple Seq2Seq model. This example shows a simplified version; real-world translation models are far more complex.

```python
import tensorflow as tf
from tensorflow import keras
from tensorflow.keras.layers import Input, LSTM, Dense
import numpy as np

# Define the model
# Example: Translating short sequences of integers

# Input data: Integer sequences representing source language words.  For demo purposes.
# Output data: Integer sequences representing target language words. For demo purposes.

# Model Parameters
num_encoder_tokens = 20  # Example vocabulary size for encoder
num_decoder_tokens = 20  # Example vocabulary size for decoder
latent_dim = 256  # Latent dimensionality of the encoding space
num_samples = 10000 # Number of samples to train on


# Define an input sequence and process it.
encoder_inputs = Input(shape=(None,), name='encoder_input')
x = keras.layers.Embedding(num_encoder_tokens, latent_dim)(encoder_inputs)
encoder = LSTM(latent_dim, return_state=True, name='encoder_lstm')
encoder_outputs, state_h, state_c = encoder(x)
# We discard `encoder_outputs` and only keep the states.
encoder_states = [state_h, state_c]

# Set up the decoder, using `encoder_states` as initial state.
decoder_inputs = Input(shape=(None,), name='decoder_input')

# We set up our decoder to return full output sequences,
# and to return internal states as well. We don't use the
# return states in the training model, but we will use them in inference.
decoder_lstm = LSTM(latent_dim, return_sequences=True, return_state=True, name='decoder_lstm')
decoder_embedding = keras.layers.Embedding(num_decoder_tokens, latent_dim)
decoder_x = decoder_embedding(decoder_inputs)
decoder_outputs, _, _ = decoder_lstm(decoder_x, initial_state=encoder_states)
decoder_dense = Dense(num_decoder_tokens, activation='softmax', name='decoder_dense')
decoder_outputs = decoder_dense(decoder_outputs)

# Define the model that will turn
# `encoder_input_data` & `decoder_input_data` into `decoder_target_data`
model = keras.Model([encoder_inputs, decoder_inputs], decoder_outputs)

# Compile the model
model.compile(optimizer='rmsprop', loss='categorical_crossentropy',
              metrics=['accuracy'])

# Generate dummy data for demonstration purposes
encoder_input_data = np.random.randint(0, num_encoder_tokens, size=(num_samples, 10))  # (num_samples, sequence_length)
decoder_input_data = np.random.randint(0, num_decoder_tokens, size=(num_samples, 10))  # (num_samples, sequence_length)
decoder_target_data = np.random.randint(0, num_decoder_tokens, size=(num_samples, 10))  # (num_samples, sequence_length)

# One-hot encode the target data
decoder_target_data = tf.keras.utils.to_categorical(decoder_target_data, num_classes=num_decoder_tokens)


# Train the model
model.fit([encoder_input_data, decoder_input_data], decoder_target_data,
          batch_size=64,
          epochs=1, #Increased epochs will increase accuracy
          validation_split=0.2)

print("Model Architecture:")
model.summary()
```

**Explanation:**

1.  **Encoder:**
    *   Takes input sequences as `encoder_inputs`.
    *   Embeds the input sequence (maps integer tokens to dense vectors).
    *   An LSTM layer processes the embedded input, producing output and hidden states (`state_h`, `state_c`). The hidden states contain information about the input sequence.
    *   We discard the `encoder_outputs` and keep only the final hidden state as the context vector.

2.  **Decoder:**
    *   Takes input sequences as `decoder_inputs`.
    *   Embeds the input sequence (maps integer tokens to dense vectors).
    *   Another LSTM layer, initialized with the encoder's hidden states, processes the embedded input. This allows the decoder to "know" what the encoder learned. `return_sequences=True` means that the LSTM outputs a sequence of hidden states (one for each time step).
    *   A Dense (fully connected) layer with a softmax activation function predicts the next token in the output sequence.

3.  **Model Definition:**
    *   The `keras.Model` combines the encoder and decoder into a single trainable unit. It takes both encoder and decoder inputs and outputs the predicted output sequence.

4.  **Training:**
    *   The model is trained using `categorical_crossentropy` loss (suitable for multi-class classification problems, which is what predicting the next word is).
    *  One-hot encoding is applied to the decoder target data for use with categorical crossentropy.

**Important Notes:**

*   This is a simplified example for demonstration purposes. In real-world applications, you would need to preprocess your data, create appropriate vocabulary mappings, and use larger datasets.
*   Attention mechanisms are often added to Seq2Seq models to improve their performance, especially for longer sequences.
*   This example uses integer sequences as input/output.  Real word translation models will use word embeddings to represent words.
