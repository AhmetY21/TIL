# Topic: Evaluation of Translation (BLEU, ROUGE)

## 1) Formal definition (what is it, and how can we use it?)

BLEU (Bilingual Evaluation Understudy) and ROUGE (Recall-Oriented Understudy for Gisting Evaluation) are automatic evaluation metrics used to assess the quality of machine-generated text, particularly in machine translation and text summarization. They operate by comparing the machine-generated text (candidate) to one or more human-written references.

**BLEU:**

*   **What is it?** BLEU primarily measures the precision of n-grams (sequences of n words) in the candidate text compared to the reference text(s). It aims to quantify how much the candidate text "agrees" with the reference(s).
*   **How can we use it?**  It calculates modified n-gram precision scores for different n-gram lengths (typically n=1 to 4).  The modification accounts for multiple occurrences of the same word in the candidate (to prevent artificially high scores).  A brevity penalty is applied to penalize candidate texts that are significantly shorter than the reference(s).  The final BLEU score is a geometric mean of the modified n-gram precision scores, multiplied by the brevity penalty. A higher BLEU score indicates better translation quality.

**ROUGE:**

*   **What is it?** ROUGE primarily measures the recall of n-grams in the candidate text compared to the reference text(s). It focuses on how much of the reference text is covered by the candidate.  Several variants of ROUGE exist, with ROUGE-N (based on n-gram recall), ROUGE-L (based on the longest common subsequence), and ROUGE-S (based on skip-bigram co-occurrence) being the most common.
*   **How can we use it?**  ROUGE-N calculates the recall of n-grams present in the reference text that are also found in the candidate text. ROUGE-L identifies the longest common subsequence between the candidate and reference texts and uses the length of this subsequence to calculate recall and precision. ROUGE-S considers the occurrence of word pairs that may not be adjacent in the text. Similar to BLEU, a higher ROUGE score indicates better quality, representing a higher degree of overlap between the candidate and reference texts.

In essence, BLEU emphasizes precision (how much the candidate agrees with the references), while ROUGE emphasizes recall (how much of the references the candidate captures). Both metrics provide a numerical score that can be used to compare different translation systems or track the improvement of a single system over time. They are used after a model generates translations for a given input text. The generated translation is then evaluated against one or more "gold standard" or "ground truth" reference translations.

## 2) Application scenario

**BLEU:** A company developing a new machine translation system wants to compare its performance against an existing commercial translation engine. They translate a large corpus of text using both systems.  BLEU can be used to automatically score the outputs of both systems against human-translated reference texts. The system with the higher BLEU score is considered to have better overall translation quality, based on precision.

**ROUGE:** A research team is working on a text summarization model. They use ROUGE to evaluate the summaries generated by their model against human-written summaries of the same documents. They can use ROUGE-L to measure how well the model captures the main points of the document (as represented in the reference summaries) based on the longest common subsequence. They could use ROUGE-2 to see how well the model gets the bigrams right.

## 3) Python method (if possible)

The `nltk` and `rouge-score` libraries in Python provide implementations of BLEU and ROUGE, respectively.

```python
from nltk.translate.bleu_score import sentence_bleu, SmoothingFunction
from rouge_score import rouge_scorer

# BLEU Example
reference = [['this', 'is', 'a', 'test', 'sentence']]  # Reference sentence (list of lists for multiple references)
candidate = ['this', 'is', 'a', 'test'] # Candidate sentence

# Smoothing function to avoid zero scores when n-grams are missing
smoothing = SmoothingFunction().method1

bleu_score = sentence_bleu(reference, candidate, smoothing_function=smoothing)
print(f"BLEU score: {bleu_score}")

# ROUGE Example
reference_text = "The cat sat on the mat."
candidate_text = "The cat was on the mat."

scorer = rouge_scorer.RougeScorer(['rouge1', 'rougeL'], use_stemmer=True)
scores = scorer.score(reference_text, candidate_text)

print(f"ROUGE scores: {scores}")
```
