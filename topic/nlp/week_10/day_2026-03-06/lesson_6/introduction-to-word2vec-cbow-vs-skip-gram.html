Topic: Introduction to Word2Vec (CBOW vs Skip-gram)

1) Formal definition (what is it, and how can we use it?)

Word2Vec is a group of related models used to produce word embeddings. These models are shallow, two-layer neural networks that are trained to reconstruct linguistic contexts of words. Word embeddings represent words as numerical vectors in a high-dimensional space. Words that share common contexts in the training corpus are located close to one another in this space.

What is it?

Word2Vec primarily comes in two flavors:

  • Continuous Bag-of-Words (CBOW): The CBOW model predicts the target word from the surrounding context words. Given a context (e.g., "the cat sat on the"), it predicts the missing word (e.g., "mat"). CBOW averages the vectors of the context words and uses the resulting vector to predict the target word.

  • Skip-gram: The Skip-gram model predicts the surrounding context words from a given target word. Given a word (e.g., "mat"), it predicts the surrounding words (e.g., "the", "cat", "sat", "on", "the"). Skip-gram tries to maximize the probability of predicting context words given the input word.

How can we use it?

Word embeddings generated by Word2Vec can be used for a variety of natural language processing tasks:

  • Semantic Similarity: Find words that are semantically similar to a given word by calculating the cosine similarity between their embeddings. For example, find words similar to "king."
  • Word Analogies: Solve analogy questions like "man is to woman as king is to ?" by performing vector arithmetic (e.g., vector("king") - vector("man") + vector("woman")).
  • Text Classification: Use word embeddings as features in text classification models. Instead of using raw words or TF-IDF features, using pre-trained or fine-tuned word embeddings often improves performance.
  • Machine Translation: Embeddings can be used to map words in one language to words in another, particularly when combined with other techniques like sequence-to-sequence models.
  • Recommendation Systems: Represent items or users as embeddings based on their interactions or attributes, enabling personalized recommendations.

2) Application scenario

Scenario: Sentiment Analysis of Customer Reviews

Imagine you're building a sentiment analysis system to analyze customer reviews for a product. You want to classify reviews as either positive, negative, or neutral.

How Word2Vec helps:

  1. Pre-training: You can pre-train a Word2Vec model on a large corpus of text data (e.g., Wikipedia, a collection of product reviews) to learn meaningful word embeddings. This captures the semantic relationships between words, even if those relationships are not explicitly present in your smaller dataset of customer reviews.

  2. Feature Extraction: You can use the pre-trained word embeddings to represent each review as a numerical vector. One common approach is to average the word embeddings of all the words in a review to create a single document embedding. Another approach would be to use the word embeddings as input into a Recurrent Neural Network or other deep learning architecture designed for sequence analysis, which allows the system to take the order of the words in a review into account.

  3. Sentiment Classification: You can then train a classifier (e.g., Logistic Regression, Support Vector Machine, Neural Network) on these document embeddings to predict the sentiment of each review.

Benefits:

  • Improved Accuracy: Word embeddings capture semantic meaning, leading to better sentiment classification compared to simpler methods like bag-of-words.
  • Handling Out-of-Vocabulary Words: Word2Vec can sometimes provide reasonable embeddings for out-of-vocabulary words if they are similar to words seen during training.
  • Transfer Learning: The pre-trained embeddings can be reused across different sentiment analysis tasks, saving time and resources.

3) Python method (if possible)

We can use the gensim library in Python to train Word2Vec models.

from gensim.models import Word2Vec
import nltk
from nltk.corpus import brown

# Download the Brown corpus (if you don't have it already)
try:
    brown.words() # Check if the corpus is already downloaded.
except LookupError:
    nltk.download('brown')

# Prepare the data: Tokenize sentences from the Brown corpus
sentences = brown.sents()

# Train the Word2Vec model
# min_count: Ignores all words with total frequency lower than this.
# window: Maximum distance between the current and predicted word within a sentence.
# vector_size: Dimensionality of the word vectors.  Common values are 100-300.
# workers: Use these many worker threads to train the model (=faster training).
model = Word2Vec(sentences, vector_size=100, window=5, min_count=5, workers=4)

# Access word vectors
vector_king = model.wv['king']
print("Vector for 'king':", vector_king)

# Find similar words
similar_words = model.wv.most_similar('king', topn=5)
print("Words similar to 'king':", similar_words)

# Save the model
model.save("word2vec.model")

# Load the model
loaded_model = Word2Vec.load("word2vec.model")

#Example usage with CBOW and Skipgram (explicitly defined)
model_cbow = Word2Vec(sentences, vector_size=100, window=5, min_count=5, workers=4, sg=0) #sg=0 indicates CBOW
model_skipgram = Word2Vec(sentences, vector_size=100, window=5, min_count=5, workers=4, sg=1) #sg=1 indicates Skipgram

# Access word vectors
vector_king_cbow = model_cbow.wv['king']
print("CBOW Vector for 'king':", vector_king_cbow)

vector_king_skipgram = model_skipgram.wv['king']
print("Skipgram Vector for 'king':", vector_king_skipgram)

# Find similar words
similar_words_cbow = model_cbow.wv.most_similar('king', topn=5)
print("CBOW Words similar to 'king':", similar_words_cbow)

similar_words_skipgram = model_skipgram.wv.most_similar('king', topn=5)
print("Skipgram Words similar to 'king':", similar_words_skipgram)

Explanation:

  1. Import Libraries: Import Word2Vec from gensim.models and nltk for data preparation.
  2. Data Preparation: The example uses the Brown corpus, a readily available text corpus. It tokenizes the corpus into sentences. You'll typically replace this step with your own text data.
  3. Model Training: Word2Vec(sentences, vector_size=100, window=5, min_count=5, workers=4) creates and trains the Word2Vec model.
    • sentences: The training data (a list of sentences, where each sentence is a list of words).
    • vector_size: The dimensionality of the word vectors (e.g., 100 dimensions).
    • window: The maximum distance between the current and predicted word within a sentence.
    • min_count: Ignores all words with total frequency lower than this.
    • workers: The number of worker threads to use for training.
    • sg: Training algorithm: 1 for skip-gram; otherwise CBOW. The latter part of the code demonstrates explicitly setting this parameter to 0 for CBOW and 1 for Skipgram.
  4. Access Word Vectors: model.wv['king'] retrieves the vector representation for the word "king".
  5. Find Similar Words: model.wv.most_similar('king', topn=5) finds the top 5 words most similar to "king" based on cosine similarity.
  6. Save and Load Model: The model.save() and Word2Vec.load() methods allow you to save and load trained models for later use. This is crucial to avoid retraining the model every time you need it.
