# Topic: Limitations of Static Word Embeddings

## 1) Formal definition (what is it, and how can we use it?)

Static word embeddings, such as those generated by Word2Vec, GloVe, or FastText, represent each word with a single, fixed vector regardless of the context in which it appears. These embeddings are typically learned from a large corpus of text by analyzing the co-occurrence patterns of words. The goal is to capture semantic and syntactic relationships between words, where semantically similar words are located closer to each other in the embedding space.

**What is it?** A fixed, pre-trained vector representation for each word in a vocabulary.

**How can we use it?**
*   **Semantic similarity:** Find words with similar meanings by calculating cosine similarity between their embeddings.
*   **Analogy completion:** Solve analogy problems like "man is to king as woman is to ?" by vector arithmetic (e.g., vector(king) - vector(man) + vector(woman)).
*   **Feature representation in downstream tasks:** Use pre-trained word embeddings as features in tasks like text classification, sentiment analysis, and machine translation. The word embeddings can be used as initial weights for an embedding layer in a neural network, or can be kept fixed and treated as external feature input.
*   **Word clustering:** Group words with similar meanings based on their embeddings.

The primary limitation is the **one-to-one mapping** between a word and its embedding. This means that words with multiple meanings (polysemy) or different usages depending on the context cannot be accurately represented. A word like "bank" (river bank vs. financial institution) will have the same embedding regardless of the context, leading to ambiguity and potentially hindering the performance of downstream tasks. Similarly, subtle nuances in meaning or sentiment are lost because the context is not taken into account.

## 2) Application scenario

Consider a sentiment analysis task where we want to determine the sentiment of a movie review. Suppose the review contains the sentence: "The bank was crowded, but the performance was surprisingly good."

Using static word embeddings, the word "bank" will have a single vector representation. This representation is likely a weighted average of the different meanings of "bank" present in the training corpus used to generate the embeddings. As a result, the embedding for "bank" might not accurately reflect its usage in this particular context (a place of business), potentially affecting the overall sentiment analysis accuracy if the system relies heavily on the word embeddings for sentiment scoring. The ambiguity introduced by the static representation could mislead the sentiment analysis model, especially if the model is trained on a different dataset where "bank" is primarily used to refer to a riverbank.

Another scenario is machine translation. Translating the sentence "I went to the bank" without knowing which sense of "bank" is intended could lead to an incorrect translation in a language where the two meanings have distinct words.

## 3) Python method (if possible)

The following code demonstrates the issue with polysemy using pre-trained GloVe embeddings and cosine similarity.  We'll calculate the similarity between "bank" and two contextually different words: "river" and "finance."

```python
import numpy as np
from sklearn.metrics.pairwise import cosine_similarity
import gensim.downloader as api

# Load pre-trained GloVe embeddings
glove_model = api.load("glove-wiki-gigaword-100")

def similarity(word1, word2):
    try:
        embedding1 = glove_model[word1]
        embedding2 = glove_model[word2]
        return cosine_similarity([embedding1], [embedding2])[0][0]
    except KeyError:
        return "Word not in vocabulary"

# Calculate cosine similarity between "bank" and "river"
similarity_river = similarity("bank", "river")
print(f"Similarity between 'bank' and 'river': {similarity_river}")

# Calculate cosine similarity between "bank" and "finance"
similarity_finance = similarity("bank", "finance")
print(f"Similarity between 'bank' and 'finance': {similarity_finance}")

# Observe the similarities.  Ideally, context-aware embeddings would show a higher similarity
# between "bank" and "river" when the sentence contains the word "river", and higher similarity with
# "finance" when the sentence contains financial terms.  Static embeddings don't provide this context.
```

The output will show that the cosine similarity between "bank" and "river" and "bank" and "finance" are likely to be relatively similar, despite the different contexts. This highlights the limitation of static word embeddings in capturing context-dependent meanings. If we had context-aware embeddings, we would expect a larger disparity between the two similarity scores, depending on the surrounding words in the sentence.
